# Double Copy Paste

A native macOS menu-bar utility that keeps a running history of your
clipboard, so copying something new never permanently loses what was there
before. See [functionality.md](functionality.md) for the product spec and
[implementation-plan.md](implementation-plan.md) for the architecture and
phased build plan.

## Requirements

- macOS 13 Ventura or later
- Xcode 15 or later
- [XcodeGen](https://github.com/yonaskolb/XcodeGen) (`brew install xcodegen`) — only needed if you change `project.yml` and need to regenerate `DoubleCopyPaste.xcodeproj`; not needed just to build/run.

## Status

Early scaffolding in progress — see `implementation-plan.md` for the
phased build order. Each phase is landed as its own commit.

## Project Structure

```
App/                  # App target: @main entry, AppDelegate, Info.plist, Assets.xcassets
Packages/
  DCPModel/            # ClipboardEntry model + ClipboardHistoryStore (pure Swift)
  DCPClipboard/         # NSPasteboard polling watcher + restore logic
  DCPUI/                 # SwiftUI views (menu bar dropdown)
project.yml            # XcodeGen spec — source of truth for the .xcodeproj
```

`DoubleCopyPaste.xcodeproj` is generated by XcodeGen from `project.yml`
and is checked into the repo so it can be opened directly; regenerate it
with `xcodegen generate` after editing `project.yml` (e.g. adding new
source files or packages).

## Building & Running

1. Open `DoubleCopyPaste.xcodeproj` in Xcode.
2. Select the `DoubleCopyPaste` scheme and press ⌘R.
3. The app has no Dock icon or window — look for its icon in the menu bar
   at the top of the screen.

To build from the command line instead:

```sh
xcodebuild -project DoubleCopyPaste.xcodeproj -scheme DoubleCopyPaste \
  -configuration Debug -destination 'platform=macOS' build
```

Each local package under `Packages/` can also be built/tested
independently without opening Xcode:

```sh
cd Packages/DCPModel && swift test
```
